# Libraries

## 1. Overview

1. Defines all technologies and libraries available for use in the project.
2. Provides a brief description of the role each library serves.

## 2. Core Principles

1. Each library has a clearly defined purpose.
2. Only approved technologies listed here should be used across the codebase.
3. Libraries should not overlap in responsibility.

## 3. Rules

### 3.1 Available Technologies

1. Vue
   1. Core reactive framework and application runtime.
2. Quasar
   1. UI framework providing components, theming, and build tooling.
3. Pinia
   1. Centralized state management solution for Vue.
4. Vue Router
   1. Controls navigation, routing, and view composition.
5. Axios
   1. HTTP client for remote API communication.
6. TypeScript
   1. Static type system used across the project.
7. ESLint
   1. Linting tool enforcing code quality and conventions.
8. Prettier
   1. Code formatter ensuring consistent source formatting.
